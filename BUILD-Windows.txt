Building QMETIS requires CMake 3.10 or newer, found at
http://www.cmake.org/. CMake generates Visual Studio project files,
which then can be built using Visual Studio. There are two ways to
generate visual studio files: using the command line and using the
CMake GUI.

Using the command line
----------------------

Open the command prompt and cd to the QMETIS source directory. Run

    > cmake --help

and look at the list of generators for the installed Visual Studio version.
After you have found the appropriate generator, run, for example:

    > .\vsgen -G "Visual Studio 17 2022" -A x64 -DSHARED=ON -DQMETIS_BUILD_PROGRAMS=OFF -DIDXTYPEWIDTH=64 -DREALTYPEWIDTH=32

to generate the project files. The project files will be placed in
build\windows. Use 32 or 64 for each width. These values configure the
generated public header; do not edit include\metis.h manually. Command-line
programs default to OFF with MSVC because DLL packages only require the library.

Using the CMake GUI
-------------------

It is also possible to use the CMake GUI, distributed with CMake. To
do this, open the CMake GUI, and browse to the location of QMETIS'
source with the "Browse Source" button. You can also change the binary
directory. Set SHARED=ON and configure IDXTYPEWIDTH and REALTYPEWIDTH
to either 32 or 64. This is where the Visual Studio project files will be
placed. Click "Generate" to select the correct visual studio version
and build the project files.

Using the VS project files
--------------------------

The Visual Studio project will be called QMETIS.sln. Open it in Visual
Studio. If the configuration is not already "Release", set it to
"Release". Type F7 to build. The QMETIS library will be in
<BINARY_DIR>\libmetis\Release and the executable programs will be in
<BINARY_DIR>\programs\Release. (<BINARY_DIR> will be build\windows if
you used the command line or whatever you choose if using the CMake
GUI.)
